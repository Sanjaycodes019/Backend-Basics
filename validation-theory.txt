INPUT VALIDATION - QUICK REVISION
==================================

Why Validate?
-----------
• Prevent invalid data in DB
• Protect against injection
• Clear error messages for users
• Reduce bugs

Joi Basics
----------
• npm install joi
• Schema-based validation
• Chain methods for rules

Common Validations
------------------
Joi.string().min(3).max(30).required()
Joi.string().email()
Joi.number().integer().min(0).max(100)
Joi.boolean()
Joi.date()
Joi.array().items(Joi.string())

Schema Example
--------------
const schema = Joi.object({
    name: Joi.string().min(2).required(),
    email: Joi.string().email().required(),
    age: Joi.number().min(0).optional(),
    role: Joi.string().valid('user', 'admin').default('user')
})

Validate
--------
const { error, value } = schema.validate(data)
if (error) return res.status(400).json({ error: error.details[0].message })

Express Middleware
------------------
const validate = (schema) => (req, res, next) => {
    const { error, value } = schema.validate(req.body)
    if (error) return res.status(400).json({ error: error.message })
    req.body = value
    next()
}

app.post('/users', validate(userSchema), createUser)

Query Validation
----------------
const querySchema = Joi.object({
    page: Joi.number().integer().min(1).default(1),
    limit: Joi.number().integer().min(1).max(100).default(10)
})

Custom Messages
---------------
Joi.string().min(3).message('Name must be at least 3 chars')

Checklist
---------
□ Validate all inputs (body, query, params)
□ Use specific error messages
□ Set defaults where appropriate
□ Sanitize after validation
□ Don't trust client data
